¿Qué es Tailwind CSS y por qué usarlo?

Tailwind CSS es un framework utility‑first: provee clases pequeñas y composables para construir interfaces rápido sin salir del HTML.

Para prototipos rápidos (CDN):

  • Sin build ni instalación: una sola línea y a trabajar
  • Iteración veloz: cambiar diseño ≈ cambiar clases
  • Diseño consistente con sistema de escalas

Ojo (CDN no es para producción):

  • Descarga grande (sin “purge” de clases no usadas)
  • Sin @apply ni plugins complejos
  • Perfecto para clases, demos y enseñanza

Instalación con CDN

Estructura mínima:

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1" /> <script src="https://cdn.tailwindcss.com"></script> </head> <body class="bg-gray-50"> <h1 class="text-3xl font-bold text-blue-600">¡Hola Tailwind!</h1> </body> </html>

Abre el archivo en el navegador. Sube/simplifica el diseño cambiando clases.

Para probar online: Tailwind Play.

Utility‑first: leer y combinar clases

<div class="p-6 bg-white rounded-xl shadow-md flex items-center space-x-4"> <div class="h-12 w-12 rounded-full bg-blue-500"></div> <div> <h3 class="text-lg font-semibold">Tarjeta</h3> <p class="text-gray-600">Construida solo con utilidades.</p> </div> </div>
  • p-6: padding
  • rounded-xl: borde redondeado
  • shadow-md: sombra
  • flex items-center space-x-4: layout y separación horizontal

Escalas de diseño (valores comunes)

Espaciado (rem ≈ px):

0 → 0px | 1 → 0.25rem (4px) | 2 → 0.5rem (8px) | 3 → 0.75rem (12px) | 4 → 1rem (16px) 6 → 1.5rem (24px) | 8 → 2rem (32px) | 10 → 2.5rem (40px) | 12 → 3rem (48px)

Colores (ejemplos):

text-gray-700 · bg-indigo-50 · border-red-300 · bg-blue-600 hover:bg-blue-700

Tipografía (tamaños):

text-xs · text-sm · text-base · text-lg · text-xl · text-2xl · text-3xl · text-4xl

Contenedores y anchuras

<div class="container mx-auto px-4">...</div> <div class="max-w-md mx-auto bg-white p-6 rounded-xl">Caja centrada</div> <div class="w-full md:w-1/2">50% en md+</div>
  • container: ancho máximo por breakpoint
  • mx-auto: centra horizontalmente
  • max-w-*: controla el ancho máximo del bloque

Tipografía rápida

<h1 class="text-3xl md:text-4xl font-bold">Titular</h1> <p class="text-gray-700 leading-relaxed">Párrafo con interlineado amable.</p> <p class="text-sm text-gray-500 italic">Bajada / copete.</p> <p class="text-base md:text-lg text-justify">Texto justificado.</p>
  • leading-*: interlineado
  • tracking-*: espaciado de letras
  • text-left/center/right/justify

Colores, fondos y bordes

<div class="bg-white border border-gray-200 rounded-lg p-6"> <h3 class="text-blue-600 font-semibold">Caja</h3> <p class="text-gray-700">Texto</p> </div>
text-red-600 · bg-emerald-50 · border-yellow-300 · rounded-full · shadow-lg

Espaciado y separación

p-4 · px-6 · py-2 · m-4 · mt-8 · -mx-2 space-y-3 (hijos en columna) · space-x-4 (hijos en fila) gap-2 · gap-4 · gap-6 (para grid/flex)

Usa space-* para listas/columnas y gap para grillas o flex.

Layout con Flexbox

<div class="flex items-center justify-between gap-4"> <div class="flex-1">Izquierda</div> <button class="bg-blue-600 text-white px-4 py-2 rounded">Acción</button> </div>
flex · flex-col · items-center · justify-center · flex-wrap · basis-1/2 · grow

Layout con Grid

<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-6"> <article class="bg-white p-4 rounded-xl shadow">Nota 1</article> <article class="bg-white p-4 rounded-xl shadow">Nota 2</article> <article class="bg-white p-4 rounded-xl shadow">Nota 3</article> </div>
grid-cols-2 · grid-cols-12 · col-span-3 · auto-rows-fr

Responsive con prefijos

<!-- Mobile first --> <div class="p-4 md:p-8 lg:p-12">…</div> <button class="w-full md:w-auto">Botón</button> <nav class="flex flex-col md:flex-row gap-2 md:gap-6">…</nav>

Breakpoints por defecto: sm (640), md (768), lg (1024), xl (1280), 2xl (1536).

Estados y variantes

<button class="bg-blue-600 hover:bg-blue-700 active:bg-blue-800 focus:ring-2 focus:ring-blue-400 disabled:opacity-50 disabled:cursor-not-allowed text-white px-4 py-2 rounded" disabled>Acción</button>

Combina hover:, focus:, active:, disabled:, etc.

Dark Mode (CDN)

Config rápida:

<script src="https://cdn.tailwindcss.com"></script> <script> tailwind.config = { darkMode: 'class' } </script>

Uso:

<html class="dark"> <body class="bg-white text-slate-900 dark:bg-slate-900 dark:text-slate-100">...</body> </html>

Alterna la clase dark en <html> con JS para cambiar el tema.

Componente: Botones

<button class="inline-flex items-center gap-2 bg-blue-600 hover:bg-blue-700 text-white px-5 py-2.5 rounded-lg shadow"> <span>Primario</span> </button> <button class="ml-3 inline-flex items-center gap-2 border border-gray-300 hover:bg-gray-50 text-gray-700 px-5 py-2.5 rounded-lg"> Secundario </button>

Reutiliza clases para variantes: color, borde, tamaño, iconos.

Componente: Card de noticia

<article class="bg-white rounded-xl shadow overflow-hidden"> <img class="w-full h-48 object-cover" src="https://picsum.photos/800/400" alt="" /> <div class="p-6"> <div class="text-sm text-gray-500">Política · 2 min</div> <h3 class="mt-1 text-xl font-semibold">Título de la nota</h3> <p class="mt-2 text-gray-700">Resumen breve del contenido...</p> <div class="mt-4 flex items-center justify-between"> <a class="text-blue-600 hover:underline" href="#">Leer más</a> <span class="text-xs bg-gray-100 text-gray-700 px-2 py-1 rounded">Última hora</span> </div> </div> </article>

Componente: Navbar responsive

<header class="bg-white border-b"> <div class="max-w-6xl mx-auto px-4 h-16 flex items-center justify-between"> <a class="font-bold" href="#">Medio</a> <nav class="hidden md:flex items-center gap-6"> <a class="text-gray-700 hover:text-blue-600" href="#">Portada</a> <a class="text-gray-700 hover:text-blue-600" href="#">Secciones</a> <a class="text-gray-700 hover:text-blue-600" href="#">Opinión</a> </nav> <button class="md:hidden p-2" aria-label="Abrir menú">☰</button> </div> </header>

Con JS mínimo, alterna un menú móvil usando hidden/block.

Patrón: Artículo periodístico

<main class="max-w-3xl mx-auto px-4 py-8"> <h1 class="text-3xl md:text-4xl font-bold">Título de la nota</h1> <p class="mt-2 text-gray-500">Autor · Fecha · Sección</p> <img class="mt-6 rounded-lg w-full" src="https://picsum.photos/1200/600" alt="" /> <article class="prose prose-gray max-w-none mt-6"> <p>Párrafos del artículo...</p> </article> </main>

Para tipografía rica, puedes usar prose con el plugin Typography (no disponible en CDN puro). Alternativa: clases utilitarias por párrafo.

Formularios accesibles

<label class="block text-sm font-medium text-gray-700" for="email">Email</label> <input id="email" type="email" class="mt-1 w-full border border-gray-300 rounded-lg px-3 py-2 focus:outline-none focus:ring-2 focus:ring-blue-400" placeholder="nombre@medio.com" /> <button class="mt-3 bg-blue-600 text-white px-4 py-2 rounded hover:bg-blue-700">Enviar</button>

Alertas y badges

Personalización con tailwind.config (CDN)

Agrega color de marca y fuente:

<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600&display=swap" rel="stylesheet"> <script src="https://cdn.tailwindcss.com"></script> <script> tailwind.config = { theme: { extend: { colors: { brand: '#0ea5e9' }, fontFamily: { sans: ['Inter', 'ui-sans-serif', 'system-ui'] } } } } </script> <!-- Uso --> <h1 class="text-brand font-sans">Marca</h1>

Con CDN, la personalización es limitada pero suficiente para prototipos.

Utilidades que aceleran

shadow-sm · shadow · shadow-lg · ring-1 ring-gray-300 rounded · rounded-md · rounded-lg · rounded-full aspect-video · object-cover · overflow-hidden line-clamp-* (plugin) · truncate · break-words divide-y divide-gray-200 · space-x-* · space-y-*

Recuerda: algunos como line-clamp requieren plugins (no CDN).

Consejos, límites del CDN y recursos

Consejos para redacciones:

  • Prototipa primero, define después estilos finales
  • Usa grillas prefabricadas y tarjetas para portadas
  • Documenta clases comunes de tu marca

Limitaciones del CDN:

  • CSS pesado (no elimina clases no usadas)
  • Sin plugins avanzados ni directiva @apply
  • No recomendado para producción

Recursos:

  • Docs: tailwindcss.com/docs
  • Playground: play.tailwindcss.com
  • Componentes: Tailwind UI (pago) y comunidades open source
